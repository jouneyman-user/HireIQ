## Coding Conventions

Inferred from the M1 spec and chosen patterns. All are mandatory — see [`CONSTITUTION.md`](./CONSTITUTION.md) for rationale.

### Backend (Python)

1. **File naming:** `snake_case.py` — `health.py`, `database.py`, `init_db.py`
2. **Function naming:** `snake_case` — `health_check()`, `get_db()`, `create_all()`
3. **Import order:** stdlib → third-party → local (enforced by `ruff`)
4. **Router pattern:** Each domain = one file in `app/routers/`; registered in `app/main.py`
5. **Error handling:** Never `except: pass`; always log + return structured JSON error
6. **Async:** FastAPI routes are sync by default; use `async def` when making async I/O calls
7. **Logging:** Use Python `logging` module — not `print()`; never log credentials or PII
8. **Type annotations:** Required on all function signatures and return types
9. **Environment config:** `os.getenv("VAR", "default")` — always document in `.env.example`
10. **Module export:** No `__all__` at M1; router files export the `router` object

### Frontend (TypeScript / React)

1. **File naming:** `PascalCase.tsx` for components, `camelCase.ts` for utilities
2. **Component naming:** PascalCase — `App`, `CandidateList`, `JobCard`
3. **Hook naming:** Prefix with `use` — `useCandidates()`, `useHealthStatus()`
4. **Import order:** React → third-party → local (enforced by ESLint)
5. **State:** `useState` + `useEffect` at M1; no global state manager yet
6. **API calls:** All calls go to `/api/*` (proxied by Vite); never hardcode `localhost:8000`
7. **Error handling:** All `fetch` calls have `.catch()` that update error state — no uncaught rejections
8. **Type annotations:** No `any` without justification comment; use Pydantic-exported types where possible
9. **Async pattern:** `async/await` with `try/catch` in `useEffect`
10. **Exports:** Default export for components, named exports for utilities

## Absolute Rules — Never Violate

From [`CONSTITUTION.md`](./CONSTITUTION.md) — the top 5 non-negotiables:

1. **`allow_origins=["*"]` is forbidden** — always use an explicit list: `["http://localhost:5173"]`
2. **Module boundaries are inviolable** — `backend/` and `frontend/` communicate only via HTTP; no cross-imports
3. **Secrets never touch source control** — not in code, not in `CLAUDE.md`, not in commit messages
4. **Migrations must be idempotent** — `make migrate` is always safe to run multiple times
5. **Standard tooling is the default** — no new tools/managers/frameworks without justification against "does pip/npm/make suffice?"

2. Create router file: backend/app/routers/{domain}.py
3. Register router in backend/app/main.py: app.include_router({domain}.router)

## Traps & Gotchas

Non-obvious things that cause mistakes in this codebase:

1. **`make dev` background process leak** — `Ctrl+C` stops Vite but leaves uvicorn running on port 8000. Next `make dev` fails. Fix: `kill $(lsof -ti:8000)` before restarting.

2. **Vite proxy strips `/api` prefix** — FastAPI routes are defined as `GET /health`, not `GET /api/health`. The rewrite rule in `vite.config.ts` removes the prefix. Writing `@router.get("/api/health")` in FastAPI will result in the effective URL being `/api/api/health`.

3. **`init_db.py` must import app.main first** — `Base.metadata.create_all()` only creates tables for models that have been imported. If you add a new model and forget to import it in `init_db.py`, the table won't be created. Always import `app.main` (or each model module explicitly) before calling `create_all`.

4. **SQLite `check_same_thread=False`** — FastAPI's async context uses multiple threads. Without this flag in the SQLAlchemy engine config, SQLite raises `SQLite objects created in a thread can only be used in that same thread`. Always present in `database.py`.

5. **`hireiq.db` relative path depends on where uvicorn runs** — `sqlite:///./hireiq.db` resolves relative to the current directory. Uvicorn must always be started from inside `backend/`. Running from the repo root creates `./hireiq.db` at the repo root (wrong location and likely gitignored in the wrong path).

6. **TypeScript errors only in build, not dev** — Vite's dev server (`npm run dev`) tolerates TypeScript errors. The production build (`npm run build`) fails on them. Always run `npx tsc --noEmit` before opening a PR.

## Glossary

Domain-specific terms used in HireIQ:

| Term | Definition |
|------|-----------|
| **M1** | Milestone 1 — the foundation scaffold: FastAPI + SQLite + React monorepo with Makefile |
| **M2+** | All milestones after M1; where authentication, production infra, and Alembic migrations begin |
| **Monorepo** | A single git repository containing both `backend/` and `frontend/` as sibling directories |
| **Vite proxy** | The `server.proxy` config in `vite.config.ts` that forwards `/api/*` requests to FastAPI, eliminating browser CORS |
| **`create_all`** | SQLAlchemy's `Base.metadata.create_all(engine)` — creates all registered model tables if they don't exist; idempotent |
| **App factory** | The pattern where `app/main.py` creates the FastAPI instance and registers all middleware and routers in one place |
| **Router** | A FastAPI `APIRouter` instance defined in `app/routers/{domain}.py`; mounted on the main app in `main.py` |
| **Conventional Commits** | Commit message format: `type(scope): description` — used throughout this repo (e.g., `docs: add M1 spec`) |
| **SemVer** | Semantic Versioning — MAJOR.MINOR.PATCH versioning scheme used by HireIQ |
| **Super Skills** | Claude Code skill framework used to generate specs and guide development in this repo |
| **PII** | Personally Identifiable Information — candidate names, emails, resumes; not present at M1 but regulated at M2+ |
| **ASGI** | Asynchronous Server Gateway Interface — the Python web server standard that FastAPI and Uvicorn implement |
| **Idempotent** | An operation that produces the same result whether run once or many times; critical for migrations |
